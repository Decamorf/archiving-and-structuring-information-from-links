# Публикация архиватора на GitHub

Схема: у вас локальная папка с программой (как сейчас) + её копия на
GitHub. После каждого обновления запускаете «publish» — на GitHub
появляется коммит и тег версии (v1.0.0.2 и т.д.). Следить за версиями и
откатываться можно во вкладке Tags / истории коммитов.

## Настройка (один раз, ~10 минут)

1. **Установите Git.**
   - Windows: https://git-scm.com/download/win — установщик, везде «Далее».
   - Linux: `sudo apt install git`
2. **Создайте репозиторий.** Зайдите на github.com → плюс справа сверху →
   **New repository** → имя, например `archiver` → выберите **Private**
   (личный, никому не виден) → Create. Ничего не добавляйте галочками.
3. **Свяжите папку с репозиторием.** Откройте командную строку в папке
   программы (в адресной строке Проводника напишите `cmd`) и выполните,
   подставив свой ник:
   ```
   git init
   git branch -M main
   git remote add origin https://github.com/ВАШ_НИК/archiver.git
   ```
4. **Первая публикация.** Запустите `publish.bat` (Linux:
   `./publish.sh`). При первом пуше Git откроет окно входа в
   GitHub через браузер — войдите один раз, дальше запомнит.

## Использование

После каждой новой версии от Claude: распаковали архив с заменой →
двойной клик по `publish.bat`. Всё. Скрипт сам берёт номер версии
из программы, делает коммит «Версия X» и тег vX.

- История: вкладка **Commits**; версии: вкладка **Tags**.
- Откатиться и посмотреть старый код: Tags → нужная версия → Browse files
  (или в самой программе, как раньше, через папку versions).
- Папки «Архив» и «Бэкапы» на GitHub НЕ загружаются (личные данные) —
  это настроено в файле `.gitignore`.
