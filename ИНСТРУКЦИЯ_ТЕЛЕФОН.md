# Доступ с телефона (Android и iPhone) — защищённый режим

## Как это устроено

Компьютер делает тяжёлую работу (Whisper, Ollama), телефон — пульт управления через браузер. С версии 1.0.0.5 доступ защищён:

- при первом запуске сервер сам создаёт **код доступа** — случайный, свой у каждой установки (проект опенсорсный, но ваш код только ваш);
- код показывается в чёрном окне сервера и хранится в файле `web_secret.txt` рядом с программой; в git-репозиторий этот файл **не попадает** (настроено в .gitignore);
- без кода не работает ничего: ни отправка ссылок, ни журнал, ни скачивание файлов;
- перебор кода бесполезен: каждая неверная попытка замедляется, после 5 неверных — блокировка на минуту;
- сервер отдаёт файлы только из папки «Архив» и шлёт браузеру защитные заголовки.

## Запуск

1. На компьютере: `python archiver_web.py` (файлы archiver.py и analyzer.py — в той же папке). Удобно сделать `start_web.bat` со строкой `python archiver_web.py`.
2. В чёрном окне появятся **адрес** (вида http://192.168.1.50:8080) и **КОД ДОСТУПА**.
3. На телефоне (та же Wi-Fi сеть): откройте адрес → введите код → «Войти». Код запоминается на этом телефоне на 30 дней.
4. По желанию: «Поделиться → На экран “Домой”» (iPhone) или меню ⋮ → «Добавить на главный экран» (Android) — будет как приложение.

## Правила безопасности (важно)

1. **НИКОГДА не пробрасывайте порт 8080 через роутер в интернет.** Локальный сервер не предназначен смотреть в открытую сеть, даже с кодом.
2. **Доступ не из дома — только через Tailscale** (бесплатно, шифрование WireGuard, настройка 5 минут):
   - установите Tailscale на компьютер (tailscale.com/download) и на телефон (из магазина приложений);
   - войдите на обоих в один аккаунт (Google/GitHub);
   - на телефоне открывайте `http://ИМЯ-КОМПЬЮТЕРА:8080` или IP вида 100.x.x.x из приложения Tailscale — работает из любой точки мира, весь трафик зашифрован, снаружи сервер не виден никому, кроме ваших устройств.
3. Код доступа можно сменить в любой момент: удалите `web_secret.txt` и перезапустите сервер — сгенерируется новый (на телефонах попросит войти заново).
4. Не публикуйте скриншоты чёрного окна сервера — там виден код.

## Для пользователей вашего опенсорс-проекта

Всё вышеописанное работает из коробки: у каждого, кто склонирует репозиторий, при первом запуске создастся собственный код, и его секреты в репозиторий не утекут. Ничего настраивать не нужно.
