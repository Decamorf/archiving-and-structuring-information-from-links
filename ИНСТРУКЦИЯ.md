# Архиватор ссылок → Markdown. Инструкция по установке и запуску

Приложение принимает ссылку на **видео** (YouTube, Rutube, VK и сотни других сайтов), **пост Telegram** (публичный канал) или **статью** — и сохраняет всё в файл `.md`:

- название файла берётся из названия видео / поста / статьи;
- внутри: автор, дата, описание, полный текст (для видео — расшифровка речи нейросетью Whisper, работает на вашем компьютере);
- в конце — список **всех найденных ссылок** (из описания, из текста, из поста);
- фото из Telegram скачиваются рядом с файлом;
- файлы автоматически раскладываются по папкам:

Файлы раскладываются **по темам** (тема определяется автоматически):

```
Архив/
├── 3D печать/
│   └── Настройка слайсера для PETG.md   (+ кадры из видео рядом)
├── IT и программирование/
│   └── Docker/                          (подтема — если включён локальный ИИ)
│       └── Деплой сайта на сервер.md
├── Технологии и гаджеты/
│   └── Советы для Steam Deck.md
└── Дом и быт/ ... и так далее
```

Внутри каждого .md: источник, автор, дата, тема, **чёткий конспект без воды**
(команды и названия кнопок — в `моноширинном` виде, ссылки — рядом с контекстом),
скриншоты моментов «нажмите сюда» с таймкодами, список всех ссылок, а полная
дословная расшифровка спрятана внизу в сворачиваемый блок.

## Шаг 1. Установить Python

### Windows
1. Откройте сайт https://www.python.org/downloads/ и нажмите жёлтую кнопку **Download Python 3.x**.
2. Запустите скачанный файл.
3. **ВАЖНО:** в самом первом окне установщика поставьте галочку **«Add python.exe to PATH»** (внизу окна). Без неё ничего не заработает.
4. Нажмите **Install Now** и дождитесь конца установки.
5. Проверка: нажмите `Win + R`, введите `cmd`, нажмите Enter. В чёрном окне напишите:
   ```
   python --version
   ```
   Если появилось что-то вроде `Python 3.12.4` — всё хорошо.

### Linux (Ubuntu / Debian / Mint)
Откройте терминал (Ctrl+Alt+T) и выполните:
```
sudo apt update
sudo apt install -y python3 python3-pip python3-tk python3-venv
```
Проверка: `python3 --version`

---

## Шаг 2. Положить файлы приложения

Создайте папку, например `Архиватор`, и положите в неё два файла:
- `archiver.py`
- `requirements.txt`

---

## Шаг 3. Установить библиотеки (один раз)

### Windows
1. Откройте командную строку **в папке с программой**: зайдите в папку `Архиватор` в Проводнике, кликните по адресной строке сверху, сотрите путь, напишите `cmd` и нажмите Enter.
2. Выполните команду (интернет обязателен, скачается ~200–300 МБ):
   ```
   pip install -r requirements.txt
   ```
3. Дождитесь надписи `Successfully installed ...`

### Linux
В терминале перейдите в папку с программой и выполните:
```
cd ~/Архиватор
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

---

## Шаг 4. Запуск

### Windows
В той же командной строке (или снова открыв `cmd` в папке):
```
python archiver.py
```
Откроется окно программы.

> Удобство: создайте в папке текстовый файл, вставьте в него строку `python archiver.py`, сохраните как `Запуск.bat` — теперь программа запускается двойным кликом по этому файлу.

### Linux
```
cd ~/Архиватор
source venv/bin/activate
python archiver.py
```

---

## Шаг 5. Как пользоваться

1. Вставьте ссылку в верхнее поле.
2. При желании смените папку для архива (по умолчанию — папка `Архив` рядом с программой).
3. Нажмите **«Сохранить»**.
4. Следите за журналом внизу — там видно, что происходит. Когда появится «ГОТОВО ✅» — файл на месте.

### Качество расшифровки видео (выпадающий список)
| Модель | Скорость | Точность | Когда выбирать |
|---|---|---|---|
| tiny | очень быстро | низкая | черновик, длинные видео |
| base | быстро | средняя | — |
| **small** | средне | хорошая | **по умолчанию, лучший баланс** |
| medium | медленно | отличная | важные видео, тихая/нечёткая речь |

## Умная выжимка, темы и скриншоты (локальный ИИ — рекомендуется)

Программа работает в двух режимах:

| | Без ИИ (сразу после установки) | С локальным ИИ (Ollama) |
|---|---|---|
| Темы папок | по словарю ключевых слов | точное определение + подтемы (например `IT и программирование/Docker`) |
| Конспект | расшифровка, очищенная от «кхм/эээ» | настоящая выжимка: суть, шаги, команды в блоках кода, таблицы |
| Скриншоты из видео | по словам-триггерам («нажмите», «вводим»...) | ИИ сам выбирает важные моменты и подписывает кадры |
| Название файла | название видео/поста | информативное название по содержанию |

Всё происходит **на вашем компьютере, без интернета и без оплаты**.

### Как включить умный режим (один раз, ~10 минут)

1. Скачайте Ollama с https://ollama.com/download (Windows — обычный установщик; Linux — команда с сайта) и установите.
2. **Убедитесь, что Ollama запущен.** На Windows после установки он обычно стартует сам — в трее (возле часов) появляется иконка ламы. Если её нет — запустите Ollama из меню «Пуск». На Linux — команда `ollama serve` (или он уже работает как служба).
3. Скачайте модель ИИ (установка Ollama её НЕ включает!). Откройте командную строку и выполните:
   ```
   ollama pull qwen2.5:7b
   ```
   Подберите модель под свой компьютер:

   | Оперативной памяти | Команда | Качество выжимки |
   |---|---|---|
   | 8 ГБ и меньше | `ollama pull llama3.2:3b` | базовое |
   | 8–16 ГБ | `ollama pull qwen2.5:7b` | хорошее (рекомендуется) |
   | 16 ГБ и больше | `ollama pull qwen2.5:14b` | отличное |

4. **Проверьте, что всё готово:**
   - команда `ollama list` должна показать скачанную модель (если список пуст — шаг 3 не выполнен);
   - откройте в браузере http://localhost:11434 — должно быть написано «Ollama is running».
5. Перезапустите архиватор — в журнале появится «Локальный ИИ найден (...)».

Анализ идёт в три этапа (тема → подробный конспект → выбор кадров), на длинном видео и слабом ПК может занять 5–15 минут — следите за журналом.

## Версия 1.0.0.1 — что нового

**Версии и бэкапы (самое главное).** В окне программы появились:
- кнопка **«💾 Бэкап»** — в любой момент создаёт полный архив в папке `Бэкапы/`: все ваши данные из «Архива» + файлы программы + все версии;
- выбор версии приложения: список установленных версий и кнопка **«Установить версию»**. Эталон — версия `1`, текущая — `1.0.0.1`. Код каждой версии хранится в папке `versions/` рядом с программой; при переключении текущая версия сохраняется автоматически, потом закройте и заново запустите программу.

**Точные скриншоты со стрелкой.** Время кадра берётся из точного таймкода фразы и привязывается к речи. Если установлена vision-модель (см. ниже) — на кадре красной рамкой и стрелкой обводится тот элемент, из-за которого кадр сделан.

**Ссылки с содержимым.** Программа заходит по упомянутым ссылкам и вставляет в раздел «Ссылки» то, ради чего их давали. Если страницу собрать не удалось — остаётся сама ссылка, как и была упомянута.

**Сжатая копия видео.** Рядом с заметкой сохраняется файл `..._видео.mp4` (H.265, 480p — обычно в 5–15 раз меньше), а в заметке раздел «Видео-файл» указывает, чем и как сжато. Отключается галочкой.

**Перевод на любой язык.** Поле «Язык заметки»: выберите из списка или впишите любой язык — конспект и тексты будут точно переведены (нужен Ollama). «Как в оригинале» — без перевода.

**Текст из видеоряда.** Если в кадре есть надписи, названия, ссылки, вставленные скриншоты — они распознаются и попадают в раздел «Текст из видеоряда» с пометкой, что это с экрана, а не из звука (нужна vision-модель).

**Иностранные слова — латиницей.** Названия программ, брендов и термины в конспекте и названии файла пишутся в оригинале (Steam Deck, Docker), без транслитерации; рядом может быть русский перевод в скобках.

### Vision-модель (для стрелок на кадрах и текста из видеоряда)
Один раз выполните:
```
ollama pull llava:7b
```
(~4.7 ГБ; программа найдёт её сама; без неё эти две функции просто пропускаются, остальное работает)

## Версия 1.0.0.2 — исправления

**Обводка на кадрах (п.2).** Модель llava слабо определяет координаты элементов — из-за этого рамок не было, а программа молчала. Теперь: в журнале всегда видно причину («модель не нашла элемент — кадр без обводки»), координаты принимаются в обоих форматах, а для обводки рекомендуется более точная модель:
```
ollama pull minicpm-v
```
Честно: локальные модели находят элемент не на каждом кадре — где не уверены, кадр остаётся чистым (это лучше, чем рамка не там).

**Текст из видеоряда (п.6).** Vision-модель читала текст плохо (фантазировала). Заменено на настоящий локальный OCR-движок RapidOCR — читает дословно и с оценкой уверенности, мусор отфильтровывается. Функция теперь **выключена по умолчанию** — включается галочкой «Текст из видеоряда (экспериментально)». Движок ставится вместе с библиотеками: `pip install -r requirements.txt`.

## Версия 1.0.0.3 — исправление установки

Пакет OCR не поддерживал Python 3.13 и ронял установку всех библиотек. Теперь OCR — необязательный: `pip install -r requirements.txt` всегда проходит, а функция «Текст из видеоряда» включается отдельной командой по желанию:

## Версия 1.0.0.4 — очередь ссылок

Больше не нужно ждать окончания обработки: вставили ссылку → кнопка **«В очередь»** (или Enter) → поле сразу очищается, можно вставлять следующую. Ссылки копятся в таблице «Очередь» со статусами (в очереди → обрабатывается… → готово ✅ / ошибка ❌) и обрабатываются по одной, автоматически, друг за другом.

- Настройки (модель, язык, папка, галочки) запоминаются для каждой ссылки в момент добавления — можно добавить одно видео с переводом, другое без.
- Кнопка **«Убрать выбранную»** — удаляет из очереди ссылку, которая ещё не начала обрабатываться.
- Обработка намеренно по одной, а не параллельно: Whisper и Ollama загружают процессор целиком, параллельность только замедлила бы всё сразу.

## Нумерация версий (SemVer)

С версии **1.1.0** проект использует стандарт Semantic Versioning — номер из трёх чисел **МАЖОР.МИНОР.ПАТЧ**:

- **ПАТЧ** (1.1.0 → 1.1.1) — исправления ошибок, ничего нового
- **МИНОР** (1.1.0 → 1.2.0) — новые функции, всё старое работает как раньше
- **МАЖОР** (1.x → 2.0.0) — изменения, ломающие совместимость (например, другой формат заметок или папок)

По номеру сразу видно, что внутри обновления. Ранние версии (1, 1.0.0.1 … 1.0.0.9) остаются в истории и папке versions под старыми именами — переключение на них работает как обычно.

## Telegram-бот (удалённый пульт)

Можно управлять архиватором из Telegram: пришлёте боту ссылку — компьютер обработает её и вернёт результат одним архивом .zip (заметка + фото/кадры), а ключевые логи продублируются прямо в чат. Работает откуда угодно, без настройки сети.

Настройка один раз:
1. В окне программы нажмите **«Настроить бота»**.
2. В Telegram напишите **@BotFather** → команда `/newbot` → задайте имя. Он пришлёт **токен** вида `123456:AA...` — вставьте в поле «Токен бота».
3. Напишите боту **@userinfobot** — он пришлёт ваш **numeric ID**. Вставьте в поле «Ваш Telegram ID».
4. Сохранить → включить галочку **«Telegram-бот»**. Статус станет «работает (@имя_бота)».
5. Напишите своему боту в Telegram любую ссылку — получите архив.

Безопасность: бот отвечает **только вашему ID**, чужие сообщения игнорируются. Токен и ID хранятся локально в `bot_config.json` и на GitHub не попадают. Компьютер должен быть включён и программа запущена (галочка активна).

## Автообновление

Приложение само проверяет наличие новой версии на GitHub при запуске (тихо, в фоне) и по кнопке **«Обновления»** в окне.

- Если вышла новая версия — программа сообщит, покажет список изменений и предложит скачать установщик. Согласитесь — она скачает `Архиватор_setup.exe` и запустит его; дальше обычный мастер поставит новую версию поверх, ваши данные (папка «Архив») не тронутся.
- Если обновлений нет, кнопка «Обновления» просто напишет «У вас последняя версия».
- Проверка ходит только на GitHub Releases вашего репозитория; ничего не устанавливается без вашего подтверждения.

Чтобы автообновление работало у пользователей, каждую новую версию выкладывайте как **Release** на GitHub с тегом (v1.5.0 и т.д.) и прикреплённым установщиком `Архиватор_setup.exe`.

## Instagram

Программа умеет сохранять посты, карусели и Reels:
- **фото-посты** — скачиваются все фото + подпись + ссылки из неё;
- **видео и Reels** — речь расшифровывается Whisper, текст попадает в файл.

Instagram активно сопротивляется скачиванию, поэтому у программы три способа получить пост: основной (instaloader), запасной (yt-dlp) и запасной с куки вашего браузера. Если пост не скачался, действуйте по порядку:

0. Обновите библиотеки — Instagram часто меняется, и это лечит большинство ошибок:
   ```
   pip install -U instaloader yt-dlp
   ```

Если ошибка про вход в аккаунт осталась:
1. Один раз выполните в командной строке / терминале:
   ```
   instaloader --login ВАШ_ЛОГИН
   ```
   (введите пароль, сессия сохранится на компьютере; команда появляется автоматически после установки библиотек из шага 3);
2. Впишите этот же логин в поле **«Логин Instagram»** в программе;
3. Повторите сохранение ссылки.

Ещё один рабочий вариант без команд: откройте instagram.com в Chrome и войдите в свой аккаунт — при ошибке основного способа программа автоматически попробует использовать куки браузера.

Используйте свой обычный аккаунт аккуратно: при очень частом массовом скачивании Instagram может временно ограничить доступ.

## Важно знать

- **Первый запуск расшифровки** скачает модель Whisper (~500 МБ для small) — нужен интернет. Дальше она работает без сети.
- Расшифровка идёт на процессоре: видео на 30 минут обычно обрабатывается за 5–10 минут (зависит от компьютера).
- Telegram: работают только ссылки на посты **публичных** каналов вида `https://t.me/канал/123`. Приватные каналы недоступны.
- Программа во время работы может «подвиснуть» визуально — это нормально, смотрите журнал.

## Если что-то не работает

| Ошибка | Что делать |
|---|---|
| `python не является внутренней командой` (Windows) | Переустановите Python с галочкой «Add to PATH» |
| `pip не найден` | Используйте команду `python -m pip install -r requirements.txt` |
| Ошибка при скачивании видео | Обновите yt-dlp: `pip install -U yt-dlp` (сайты часто меняются) |
| `No module named tkinter` (Linux) | `sudo apt install python3-tk` |
| Пост Telegram не скачивается | Проверьте, что канал публичный и ссылка ведёт на конкретный пост |
| «Локальный ИИ (Ollama) не найден» | 1) Запущен ли Ollama? (иконка в трее / `ollama serve`) 2) Скачана ли модель? `ollama list`; если пусто — `ollama pull qwen2.5:7b` 3) В браузере http://localhost:11434 должно быть «Ollama is running» |
| Выжимка слишком короткая или странная | Возьмите модель побольше (см. таблицу выше), например `qwen2.5:14b` |
| Ошибка при скачивании из Instagram | `pip install -U instaloader yt-dlp`, войдите в instagram.com в Chrome, при необходимости `instaloader --login ВАШ_ЛОГИН` |
